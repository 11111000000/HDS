# HDS — 叙事与哲学：形与空、事实与慈悲

## 形与空
全息图不是为文档而文档，它是系统的“骨骼”：边界（SURFACE）、法律（不变式）、典范（确定性/版本），以及一条活的路径。空，是演化的空间：FLUID 区与桩件。无形先肉，必生债；无现实先典范，会固化错误。平衡之道：先形（Surface/不变式），再证（场景），后肉（代码）。

## 无为（Wu Wei）作为工程纪律
避免引发连锁重写的改动。先 Surface/Proof，后代码。不要与工具（LLM）的本性对抗：给出清晰河岸（HOLO/SURFACE/tests），而非无边自由。

## 以可复现事实为真
项目之真不在“感觉对”，而在可复现的测试或反例（Refutation）。不变式是法律（意图），测试是判例（事实）。二者合一，抵抗语义漂移。

## 双速世界
探索要快，但不侵蚀河岸。Frozen/Core 的变更要慢，有事实与迁移作背书。Fluid/Periphery 的变更要快，但不能破坏典范。边界体现对未来的关怀。

## 向未来的慈悲
HOLO 与 SURFACE 是写给未来的自己与同伴（含 LLM）的信。它们提供方位：看哪里、动什么、不该破什么、如何验证。这是伦理：更少惊喜，更多可解释性。

## 心智建议
- 写代码前先答：表面会变什么？我们如何证明？
- 若 LLM “吃力”，不是它的错：那是认知负债的信号。简化形。
- 若在争论品味，请回到不变式。若尚无不变式，先写法律，再谈测试。
