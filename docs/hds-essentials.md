HDS Essentials — суть голографической разработки (нормативно и кратко)

Что такое HDS
Голографическая разработка — это способ держать “целое в малом”: в любой момент у проекта есть минимальный срез (голограмма), по которому можно восстановить публичный смысл, законы и проверку их выполнения. Голограмма не бюрократия, а карта реальности: Manifest (HOLO), Contract Surface, Proof (тесты), один вертикальный сценарий.

Ключевые понятия
- HOLO (Manifest): один живой файл (HOLO.md) — текущая стадия, 5–15 инвариантов (законы), ключевые решения (Decisions) с Exit-criteria. Если код противоречит HOLO — баг в коде; если HOLO устарел — проект теряет форму.
- Contract Surface (SURFACE.md): внешний контракт (API/форматы/схемы/CLI/события/ops). Каждый пункт помечен стабильностью: [FROZEN] — канон, меняется только через Pressure+Proof+миграцию; [FLUID] — зона эксперимента, не ломает Frozen.
- Invariants & Proof: инварианты — смысловые законы (“что нельзя ломать”); Proof — тесты (contract/scenario/property/lint), которые делают законы проверяемыми в CI. Закон (инвариант) — намерение; тест — его судебная практика.
- Core/Periphery: ядро смысла (Core) не зависит от IO; периферия (Periphery) — адаптеры/интеграции. Это делает систему антихрупкой: мир меняется, ядро остаётся.
- Two-Speed: быстрый контур (Fluid/Periphery) и медленный контур (Frozen/Core/Stable). Скорость без расползания.

Стадии (режимы зрелости, без тяжелых процессов)
- Context: черновой Surface и инварианты; решения Draft.
- Skeleton: формы (типы/сигнатуры/схемы) существуют, сборка проходит.
- Reality Check: есть вертикальный сценарий (e2e на моках допустим).
- Canon: канон ID/версий/сериализации; property/roundtrip; часть решений Frozen.
- Core: ядро смысла покрыто golden/scenario; периферия не меняет смысл ядра.
- Integrate/Ops: интеграции/миграции/наблюдаемость/безопасность согласованы с Surface.

Четыре закона (аксиомы) HDS
- Surface First: меняется внешний смысл — сначала SURFACE.md, затем код.
- Frozen Requires Proof: каждый [FROZEN] имеет проверяемые тесты (в CI).
- One Change, One Intent: одно изменение — одна цель.
- Pressure for Frozen: Frozen меняется только под давлением (Bug/Feature/Debt/Ops) с явным Proof и, при необходимости, миграцией.

Протокол изменения (помещается в PR)
Intent — одна фраза; Pressure — Bug/Feature/Debt/Ops; Surface impact — какие пункты SURFACE и их стабильность; Proof — какие тесты подтвердят. Если затронут [FROZEN], добавьте Migration/Compatibility note.

Роль LLM
LLM действует в рамках голограммы: читает HOLO.md и SURFACE.md, меняет сначала Surface (если смысл меняется), затем добавляет/правит тесты, затем код. Запрещено менять Frozen без Pressure и Proof. Рекомендуемый порядок запроса: Intent → Pressure → Surface impact → Proof.

Критерий голографичности (Holo)
Состояние репозитория считается голографичным, если: HOLO.md существует и содержит ≥5 инвариантов; SURFACE.md существует и имеет ≥1 [FROZEN] с валидным Proof; есть ≥1 вертикальный сценарий (tests/scenario/*); инструменты проверки проходят (tools/holo-verify.sh). См. docs/hds-spec.md и docs/hdf-compact-spec.md для подробностей; начните с docs/quickstart-5min.md.
