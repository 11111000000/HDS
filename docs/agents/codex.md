# Codex Analysis

## Typical Surfaces

- Repo-level instruction files such as `CODEX.md` or equivalent loader-aware entrypoints.

## HDS Fit

Codex should be treated like the other runtimes: a small entry file that points to shared HDS guidance, not a copy of the full protocol.

## Dialectical Risk

- If the Codex entrypoint contains the whole method, it becomes a second canonical source.

## HDS Recommendation

- Keep `CODEX.md` thin.
- Point to the shared loader/core.
- Preserve the same two-cycle order as the other runtimes.
