HDS Overview — весь смысл в малом, без бюрократии (нормативно-прикладное введение)

Задача
Современная разработка, особенно с LLM, ускорилась, но потеряла форму: публичный смысл расползается, тесты не фиксируют намерение, патчи становятся широкими. HDS (Holographic Development Specification) предлагает простой каркас, где “целое видно в части”: на любом этапе существует минимальная голограмма проекта — манифест, поверхность, доказательства и один вертикальный сценарий.

Ключевые понятия
Голограмма — минимальный срез, из которого можно восстановить целое. В нашем случае это HOLO.md (этика и законы), SURFACE.md (границы и обещания), Proof (тесты) и один вертикальный сценарий (жизнеспособность формы).
Contract Surface — реестр внешних обещаний (API, форматы, схемы, события, CLI, ops). Каждый элемент помечен стабильностью: [FROZEN] — канон совместимости; [FLUID] — зона эксперимента. Замороженное меняется только под давлением факта (Pressure) и с доказательством (Proof).
Core/Periphery — смысл (Core) не зависит от IO; периферия — адаптеры/интеграции. Это даёт антихрупкость: мир может меняться, ядро остаётся.
Decisions — ключевые выборы с Exit-criteria (условием пересмотра). Они не догма, а договор, пересматриваемый фактами.

Двухскоростной контур
Fluid — быстрые изменения без ломки Frozen. Frozen/Core — медленные изменения под давлением с тестами и миграциями. Это сохраняет скорость и идентичность одновременно.

Как это превращается в практику
- Manifest (HOLO.md) — один живой файл: текущая стадия, 5–15 инвариантов, ключевые решения (с Exit), Reality Check-сценарий.
- Surface (SURFACE.md) — короткий реестр контрактов с пометкой стабильности и Proof-ссылками.
- Tests — контрактные для Frozen (детерминизм, roundtrip, совместимость) и хотя бы один вертикальный сценарий (e2e или на моках).
- PR протокол — четыре строки: Intent, Pressure, Surface impact, Proof. Если Frozen — миграция/совместимость.

Стадии (режимы зрелости, без ритуалов)
Context → Skeleton → Reality Check → Canon → Core → Integrate → Ops. Стадия фиксируется в HOLO.md и определяет, что считается “достаточной голограммой” прямо сейчас.
